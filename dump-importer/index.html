This is an import tool for dumps from chrome://webrtc-internals and rtcstats. See this blog post for a lengthy description of what it does and how to interpret some of the data.

Try rtcstats.com which adds a lot to the raw data visualization you find here:

  • Reduces the time you'll glare at numbers and graphs from hours to minutes.
  • Adds media quality evaluation algorithms (think MOS for audio and video).
  • Makes important observations for you, like a WebRTC expert.
  • Finds root causes of issues with the help of AI.
Did we mention it is free to use?


. This generates a copy of the webrtc-internals or rtcstats dump with IP addresses obfuscated in the following events:
  • candidate lines in the SDP in setLocalDescription, setRemoteDescription et al
  • ICE candidates (with the exception of TURN server / relay addresses)
  • statistics
The obfuscation replaces the last part of IPv4 and the last six parts of IPv6 with an 'x'. This removes some of the privacy concern about uploading this e.g. to the Chromium issue tracker.
